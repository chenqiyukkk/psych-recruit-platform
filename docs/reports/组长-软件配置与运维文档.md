# 软件配置与运维文档（心理学被试招募平台）

> 说明：本文档为 Markdown 草稿版，供组内同步与后续导出 PDF 使用。  
> 技术栈：Spring Boot 2.7、MySQL 8.0、Redis（可选）、Nginx、Docker（可选）。

## 1. 开发环境配置

### 1.1 JDK 11 安装

要求：后端使用 JDK 11 编译与运行。

检查：
```bash
java -version
javac -version
```

### 1.2 Maven 配置（推荐用 IntelliJ IDEA 管理）

项目后端为 Maven 工程（`backend/pom.xml`）。

方式 A：IDEA 自动管理（比较推荐）
- 打开仓库根目录
- 若提示导入 Maven，选择 Import/Load
- 右键 `backend/pom.xml` → `Add as Maven Project`（如果没有自动识别）

方式 B：本机安装 Maven 后运行
```bash
mvn -v
mvn -f backend/pom.xml test
```

### 1.3 MySQL 安装与配置

要求：MySQL 8.0。

1）创建数据库：
```sql
CREATE DATABASE IF NOT EXISTS psych_recruit_platform
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_0900_ai_ci;
```

2）执行建表脚本：
- 脚本路径：`database/schema/schema.sql`

### 1.4 Redis 安装（可选）

当前阶段可先不启用 Redis；后续若需验证码/缓存/限流，可补充 `spring-boot-starter-data-redis` 依赖与配置。

## 2. 启动与运行

### 2.1 后端启动（本地）

方式 A：IDEA 运行
- 运行入口：`backend/src/main/java/com/project/PsychRecruitPlatformApplication.java`

方式 B：命令行
```bash
mvn -f backend/pom.xml spring-boot:run
```

默认端口：
- `8080`

Swagger（若启用成功）：
- `http://localhost:8080/swagger-ui.html`

### 2.2 关键配置项说明

配置文件：
- `backend/src/main/resources/application.yml`

主要配置项：
- `spring.datasource.*`：数据库连接（url/username/password）
- `spring.jpa.hibernate.ddl-auto`：建议 `validate`（以脚本为准，避免自动改表）
- `app.jwt.secret`：JWT 密钥（需至少 32 字符）
- `app.jwt.expiration-ms`：token 过期时间

## 3. 部署方案（Docker 方式，可选）

> 说明：本仓库当前仅提供文档草稿；如需 Docker 化，后续可补充 `backend/Dockerfile` 与 `docker-compose.yml`。

建议组件：
- 后端服务容器（Spring Boot jar）
- MySQL 容器（或使用云数据库）
- Redis 容器（可选）
- Nginx 容器（反向代理 + HTTPS）

## 4. Nginx 反向代理示例

示例（仅示意）：
```nginx
server {
  listen 80;
  server_name example.com;

  location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}
```

## 5. 日志管理

建议：
- 生产环境将日志输出到文件并按天滚动
- 避免日志打印敏感字段（密码、token、证据链接中的隐私信息）

## 6. 数据库备份策略

建议：
- 每日全量备份（低频）+ 重要节点增量备份（按需）
- 备份文件加密存储并限制访问权限

示例：
```bash
mysqldump -uroot -p psych_recruit_platform > backup.sql
```

## 7. 监控方案

最小方案：
- 基础：CPU/内存/磁盘、进程存活、端口探活
- 应用：HTTP 5xx、接口耗时、数据库连接数

可选：
- Spring Boot Actuator + Prometheus + Grafana

## 8. 故障处理流程

1）确认故障范围（仅后端 / 数据库 / 网络 / 客户端）
2）查看应用日志与 Nginx 日志（如有）
3）检查数据库连接与慢查询
4）必要时回滚到上一版本（保留可运行构建产物与配置快照）
